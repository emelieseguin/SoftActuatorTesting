# Instrument Console prototype

**Date:** 2026-07-11
**Related todo:** `prototype-console`
**Selection outcome:** Chosen by
[`ADR 0005`](../../architecture/0005-ui-shell-evaluation.md) as the normal
shell direction; it remains demo-backed and is not production-ready.

Instrument Console is the dark, dense, dockable evaluation shell from ADR
0005. It composes the shared `PAGE_REGISTRY` pages with one deterministic
`DemoEnvironment`; it does not add hardware, persistence, or workflow content
outside those pages. The shell owns navigation, window chrome, the
persistent top connection/calibration/camera/storage/run status strip and
always-visible global Stop, and four dockable panels (Telemetry, Run
Control, Event Log, File / Context) with in-memory save/restore layout
support.

## Test plan

- Navigate with every compact left-rail action (mouse and its `Ctrl+1`…
  `Ctrl+8` shortcut) and assert the corresponding registered page becomes
  active and its action stays checked.
- Move, float, hide, and tab the four dock panels; assert `Ctrl+Shift+L`
  (save) / `Ctrl+Shift+R` (restore) round-trip the arrangement, and that
  restoring a layout on a freshly constructed window never connects the fake
  serial/camera services or starts a run.
- Enable readiness, start a demo run, and assert the global Stop button (top
  strip, always visible) is enabled only while a run is active; invoke it by
  mouse and by its `Ctrl+Shift+S` shortcut from a page other than Live Run,
  and assert the run is aborted (`RunCompletion.ABORTED`) and the Live Run
  page shows the fault presentation.
- Assert accessible names/descriptions on the status strip indicators, the
  Stop button, navigation buttons, and all four docks; confirm keyboard
  focus moves into the active page on navigation.
- Drive `run_full_workflow_demo()` (also bound to `Ctrl+Shift+W` and the
  Demo menu) and assert it walks workspace → connections → calibration →
  geometry → experiment → live run → analysis to a clean completion using
  only deterministic fake services.
- Exercise the demo-only scenario switch (the "Demo state" combo box in the
  top strip, and the per-scenario Demo-menu entries) across all six explicit
  empty/loading/ready/running/completed/fault states and assert every page
  reflects the applied scenario.
- Render at 1280×720 and grab a deterministic image as a headless smoke
  test. The reference PNG is generated by
  `UPDATE_UI_REFERENCE=1 uv run pytest tests/ui/test_instrument_console.py -k screenshot`
  and is intentionally a review artifact, not a pixel-comparison baseline.

## Reference image

`instrument-console-reference.png` shows the completed **Live Run** page at
1280×720 in the offscreen Qt test environment, with the Telemetry and Run
Control docks on the right and the Event Log / File Context docks tabbed at
the bottom, after `run_full_workflow_demo()` has walked the full simulated
flow. It is refreshed only when the dedicated screenshot smoke test is
explicitly run with `UPDATE_UI_REFERENCE=1`.
