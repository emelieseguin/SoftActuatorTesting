# Experiment Studio prototype

**Date:** 2026-07-11
**Related todo:** `prototype-studio`
**Selection outcome:** Rejected as the normal shell by
[`ADR 0005`](../../architecture/0005-ui-shell-evaluation.md); retained only
through `--prototype experiment-studio` for development comparison.

Experiment Studio is the light, guided evaluation shell from ADR 0005.  It
composes the shared `PAGE_REGISTRY` pages with one deterministic
`DemoEnvironment`; it does not add hardware, persistence, or workflow content
outside those pages.

## Test plan

- Navigate with each persistent stage button, including revisiting a completed
  stage, and assert the corresponding registered page is active.
- Advance a full simulated flow and assert the visible readiness/progress
  summary changes from blocked to ready and records each completed stage.
- Start the deterministic live run, invoke the always-visible global Stop
  action (including its `Ctrl+Shift+S` shortcut), and assert clean completion.
- Assert accessible names, keyboard focus, text/glyph status labels, and
  advanced-details disclosure behavior.
- Render at 1280×720 and grab a deterministic image as a headless smoke test.
  The reference PNG is generated by
  `uv run pytest tests/ui/test_experiment_studio.py -k screenshot` and is
  intentionally a review artifact, not a pixel-comparison baseline.

## Reference image

`experiment-studio-reference.png` shows the ready **Prepare Run** stage at
1280×720 in the offscreen Qt test environment. Refresh it from the repository
root with:

```sh
UPDATE_UI_REFERENCE=1 uv run pytest tests/ui/test_experiment_studio.py -k screenshot
```

The default screenshot smoke test writes only to pytest's temporary test
location, so normal test runs never modify a review artifact.
